## 2. โครงสร้าง 4 บทบาท (Roles)

| บทบาท | สิทธิ์ | จำนวน |
|---|---|---|
| **super_admin** | ทุกอย่าง + รับแจ้งเตือน Telegram | 1 คน |
| **developer** | จัดการทุกอย่างหลังบ้าน + กำหนด role | หลายคน |
| **admin** | เพิ่ม/ลดคะแนนเท่านั้น | หลายคน |
| **member** | ใช้งานหน้าบ้าน | หลายคน |

**กติกาความปลอดภัย:** developer ตั้ง super_admin ไม่ได้ · เฉพาะ super_admin โอนตำแหน่งตัวเอง · super_admin ถูกลดโดยผู้อื่นไม่ได้

## 5. คู่มือย้ายไปเครื่องใหม่ 🖥️

### ขั้นที่ 1: ติดตั้งโปรแกรมพื้นฐาน
1. **Node.js** เวอร์ชัน 20 ขึ้นไป → https://nodejs.org (เลือก LTS)
2. **Git** → https://git-scm.com
3. **โปรแกรมแก้โค้ด** เช่น VS Code → https://code.visualstudio.com

### ขั้นที่ 2: เอาโค้ดลงเครื่อง (เลือก 1 ใน 2 ทาง)

**ทาง A — แตกไฟล์ zip (ที่แนบมา):**
1. แตก `caketruk.zip` ไปไว้ที่ต้องการ (เช่น `D:\caketruk`)
2. ไฟล์ `.env` มีอยู่ในนั้นแล้ว (มีค่าลับครบ)

**ทาง B — โหลดจาก GitHub (แนะนำ ใหม่สด):**
```bash
git clone https://github.com/warintjit/caketruk.git
cd caketruk
```
แล้ว**สร้างไฟล์ `.env` เอง** (ก๊อปจาก `.env.example` แล้วเติมค่าจาก Supabase/Google — ดูข้อ 4)

### ขั้นที่ 3: ติดตั้ง + รัน
```bash
npm install          # ติดตั้งไลบรารี (ครั้งแรกใช้เวลาสักครู่)
npm run dev          # เปิดเว็บที่ http://localhost:5173
```

### ขั้นที่ 4: ตั้งค่า git (ถ้าใช้ทาง B clone มา ข้ามได้)
```bash
git config --local user.name "Kru_Boat"
git config --local user.email "warintjit@gmail.com"
```
ครั้งแรกที่ `git push` จะมีหน้าต่างให้ล็อกอิน GitHub — ทำครั้งเดียว

> ⚠️ **ไฟล์ `.env` สำคัญมาก** — มีค่าลับ (รหัสฐานข้อมูล/key) อย่าอัปขึ้น GitHub (กันไว้แล้วใน .gitignore) และอย่าแชร์ไฟล์ zip ให้คนอื่นเพราะมี .env อยู่ข้างใน

## 7. ขั้นตอนถัดไป (เมื่อกลับมาทำต่อ)

แนะนำทำ **Phase 3 (หน้าบ้านสมาชิก)** หรือ **Phase 4 (หลังบ้านแอดมิน)** ก่อน เพราะทำได้ทันทีไม่ต้องรออนุมัติ
ส่วน **Phase 2 (LINE)** ควรเริ่ม**สมัคร LINE OA + LINE Login channel** คู่ขนานไว้ เพราะมีขั้นตอนรอ

### หมายเหตุเทคนิคที่ได้เรียนรู้ (กันลืม)
- ต่อฐานข้อมูล Supabase จากเครื่องนอกต้องใช้ **pooler** (`aws-1-ap-southeast-1.pooler.supabase.com`) ไม่ใช่ direct connection (ติด IPv6)
- หลังรัน SQL (DDL) ต้อง `notify pgrst, 'reload schema';` ให้ API เห็นตารางใหม่
- รัน migration: ใช้ไลบรารี `pg` อ่าน `SUPABASE_DB_URL` จาก `.env`
